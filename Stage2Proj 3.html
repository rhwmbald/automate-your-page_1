Stage 0: The Basics

How the Web Works

When a person goes to a web page like www.google.com, their computer sends a HTTP Request to a server.

The server finds the appropriate HTML document and sends it back to the user's computer where a web browser interprets the page and displays it on the user's screen. This video does a good job of explaining more if you can view it.

HTML

HTML, Hypertext Markup Language, documents form the majority of the content on the web.

HTML documents contain text content which describes "what you see" and CSS which describes "how it looks". This video gives a good overview if you can view it.

Tags and Elements

HTML documents are made of HTML elements. When writing HTML, we tell browsers the type of each element by using HTML tags. This video explains the distinction well if you can view it.

Why Computers are Literal - Not Stupid

Computers do only what they are told because they interpret instructions literally. This makes them very unforgiving since a small mistake by a programmer can cause huge problems in a program.

So you need to be a better programmer because Garbage in = Garbage out.

Inline vs Block Elements

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

Stage 1: Creating a Structured Document with HTML

Developer Tools (in the Browser)

There are HTML editors and CSS editors. The Documenation Object Mode (DOM) knows what to do with elements to translate them properly.

The "tree-like structure" of HTML

The "tree-like structure" comes from the fact that HTML elements can branch off. In a browser, this structure shows up as a series of nested boxes.

The relationship between indented HTML and boxes

When you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested it's corresponding "box" is.

Text Editors (for programming)

When writing code, programmers use special text editors (like Sublime Text for example). These editors make the programmer's life easier.

A computer program is used to do tasks that would require manual repetition so that it is efficient and effective. A computer will follow the instructions in a program to accomplish a specific task. There are many programs but the one we will use is called Python.

Strings and Variables are an essential part of most programming languages and is a way to pass inputs into the program. Strings hold alphanumeric characters like text='Hello' or text ='2' and are in quotes. Variables are assigned a value and the value can be reassigned. Assignment of a value in Python, happens in a line like counter = 100 were counter is assigned a number of 100.

The equal sign can mean different things. in a line of code 4-1=3 , the equal sign means the same as. However in a line of code counter=100 , the equal means take the value of.

Variables are very useful in programming. Variables are given names that tell what they do so this make the program understandable to others especially programs that have to support it later in time. They store data and allow the data to be inputs to the other places in the program.

Functions(also called Procedures)

Functions are groups of code that manipulate input to produce an output. A simple function would be to take an income and apply a tax rate to determine the amount of tax owed. The inputs would be the income and tax rate and the output would be the tax owed.

In Python, functions start with the word 'def' and the functions name with a colon afterwards. The body would be the instructions and then the required output would be returned. Using the example above we could do:

  • 1. def tax_owed(inc,tax):
  • 2.       amt=0
  • 3.       amt = inc * tax
  • 4.       return amt
  • 5. print "Tax Owed is " + str(tax_owed(1000,.1))

The function is tax_owed and it returns 100 (1000*.1). Then Python prints the string and the function's output for "Tax Owed is 100.0"

Once a function is defined, it be used over and over again - helping to reduce repetition, reduce errors, and save time. If the return was forgotten in the function then a 'None' would be the result. In Python, letter case and indentions are very important. The != means not equal and == means equal. The # comments out the line of code and three quotes will comment out multiple lines.
